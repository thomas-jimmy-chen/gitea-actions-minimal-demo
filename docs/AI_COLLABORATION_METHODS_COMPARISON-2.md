```
┌─────────────────────────────────────────────────────────────────────┐
│  方法 1: 結構化規格驅動開發                                          │
├─────────────────────────────────────────────────────────────────────┤
│  [需求收集] → [規格撰寫] → [規格審核] → [實作] → [驗證] → [交付]     │

└─────────────────────────────────────────────────────────────────────┘

│  方法 2: 迭代式探索開發                                              │
├─────────────────────────────────────────────────────────────────────┤

│       ▲                                      │                      │
│       └──────────────────────────────────────┘                      │
│  特點: 循環流程，每次迭代都可交付                                    │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│  方法 3: 過渡方法                                                    │
├─────────────────────────────────────────────────────────────────────┤

│  特點: 一次性過渡流程                                                │

│  方法 4: 混合漸進式開發                                              │
```

## 13. 提示詞模板

### 13.1 方法 1 提示詞

**開始專案**:
```
我要開發一個 [專案描述]。
需求：1. [功能1] 2. [功能2] 3. [功能3]
技術限制：語言: [...], 框架: [...]
請幫我產出需求規格文檔、API 規格、邊界值條件。
```

### 13.2 方法 2 提示詞

**開始探索**:
```
我想做一個 [模糊的想法]。
目前知道的：[...] 不確定的：[...]
請先幫我做一個最簡單的版本，我看到效果後再告訴你要怎麼調整。
```

### 13.3 方法 3 提示詞

**開始過渡**:
```
我們之前用迭代方式開發的 [專案名稱]，現在需求穩定了。
請幫我：1. 盤點功能和狀態 2. 識別技術債務 3. 整理需求規格
```

### 13.4 方法 4 提示詞

**盤點狀態**:
```
請幫我盤點目前各模組的狀態：
- [模組 A]
- [模組 B]
- [模組 C]
請分類為：🔄 探索中 / 🔄→📋 轉換中 / 📋 已穩定
```

**局部轉換**:
```
[模組名稱] 的需求已經穩定，請幫我為這個模組撰寫規格。
其他模組維持目前的開發方式。
```

## 16. 實際案例：EEBot 專案

### 16.1 專案背景

- **專案名稱**: EEBot - TronClass Learning Assistant
- **代號**: AliCorn (天角獸)
- **開發時間**: 2025-01 ~ 至今
- **採用方法**: 方法 4 (混合漸進式開發)

### 16.2 模組狀態分佈

```
📋 已穩定 (60%)          🔄→📋 轉換中 (25%)      🔄 探索中 (15%)
════════════════         ════════════════        ════════════

• src/core/              • src/services/hybrid/  • orchestrators/
• src/pages/             • exam_auto_answer.py   • captcha_ocr.py
• src/api/               • api_scanner.py        • recommender.py
• question_bank.py       • execution_wrapper.py
```

### 16.3 CLAUDE.md 實例

EEBot 的 `CLAUDE.md` 包含完整的模組狀態追蹤：

```
src/
├── core/                    📋 已穩定
├── pages/                   📋 已穩定 (POM Pattern)
├── services/                🔄→📋 轉換中
├── api/                     📋 已穩定
├── scenarios/               🔄→📋 轉換中
├── orchestrators/           🔄 探索中 (新架構)
└── utils/                   📋 已穩定
```

### 16.4 經驗總結

| 經驗 | 說明 |
|------|------|
| **核心先穩定** | `src/core/` 最早穩定，成為基礎 |
| **按需轉換** | 不強求全部轉換，需要時再正式化 |
| **文檔隨行** | 規格文檔隨模組穩定逐步補充 |
| **新功能先迭代** | `orchestrators/` 是新架構，仍在探索 |
| **狀態可視化** | 在 CLAUDE.md 中維護狀態表 |
